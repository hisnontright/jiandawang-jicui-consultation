# 检答网集萃本地检索咨询

## 核心原则

本技能用于基于内置“检答网集萃第1–140批”全文回答法律和检察相关问题。它的价值在于可溯源检索，而不是一般法律知识问答。

处理用户问题时，先检索本技能内置资料，再回答。不要凭模型记忆生成检答网观点、批次号、问题号、发布日期、专家姓名或答疑意见。

## 资料边界

- 内置全文目录：`references/batches/`
- 快速索引文件：`references/index.md`
- 通俗表达与法律术语映射：`references/legal-term-map.md`
- 全文检索脚本：`scripts/search_jiandawang.py`
- 索引生成脚本：`scripts/build_index.py`

本技能只代表“检答网集萃第1–140批”这个资料范围。它不是完整法律法规库、案例库或裁判文书库。

## 触发后必须执行的流程

### 1. 判断问题是否属于检答网集萃检索场景

遇到以下情况，使用本技能：

- 用户明确提到“检答网”“检答网集萃”“检察日报答疑”“集萃里有没有”。
- 用户咨询检察机关办案实务或检察业务疑难问题。
- 用户提到或实质涉及：认罪认罚、不起诉、抗诉、逮捕、羁押、国家赔偿、公益诉讼、社会公共利益、侦查监督、刑事执行检察、控告申诉检察、未成年人检察、民事行政检察等。

如果用户要求完整法律法规检索、案例检索或企业风险检索，而不是检答网集萃答疑，应先说明本技能资料边界；经用户同意后再改用其他检索技能或工具。

### 2. 生成检索词

如果用户已经使用具体法律概念，例如“认罪认罚”“不起诉”“抗诉”“国家赔偿”“公益诉讼”，直接把这些概念作为核心关键词进行模糊匹配。

如果用户使用通俗语言，先阅读或参考 `references/legal-term-map.md`，把通俗表达扩展为法律/检察术语，再合并用户原文一起检索。例如：

- “人被抓了，后来证据不够没起诉，但公安还在查，能不能要赔偿？”
- 扩展为：“刑事拘留、逮捕、事实不清、证据不足、不起诉、继续侦查、国家赔偿、刑事赔偿”。

术语扩展只用于检索，不得把术语映射本身当作法律结论。

### 3. 先查索引，再查全文

先使用 `references/index.md` 快速了解可能相关的批次、问题号、咨询类别和主题词。索引只用于缩小范围，不足以直接作答。

随后运行全文检索脚本。推荐命令：

```bash
python "${CLAUDE_SKILL_DIR}/scripts/search_jiandawang.py" --query "用户问题或扩展后的检索词" --top-k 8 --json --include-block
```

如果环境没有提供 `CLAUDE_SKILL_DIR`，先进入本技能目录后再运行 `python scripts/search_jiandawang.py ...`。脚本会按自身所在位置读取内置 `references/batches/*.md` 并返回结构化结果。

### 4. 读取完整问题块并判断支撑强度

检索到候选结果后，优先依据 `question_block` 或对应 Markdown 文件中的完整 `## 问题N：...` 块判断是否能支撑用户问题。不要只依赖 `excerpt` 摘要作答。

判断规则：

- 能直接回答用户问题：基于该问题块整理答复，并引用脚本返回的 `citation`。
- 只相近但不能直接支撑：说明“检索到相近内容，但未检索到能直接支撑该问题的答疑内容”，可列出相近内容的边界。
- 没有检索结果：说明“未在检答网集萃中检索到相关内容”，不得继续编造检答网观点。

## 引用格式

引用任何答疑内容时，逐条使用脚本生成的 `citation` 字段，格式必须为：

```text
检答网集萃第X批·问题Y（检察日报，YYYY-MM-DD）
```

注意：原文来源可能写作“检察日报-理论版”等，但用户要求的引用格式统一使用“检察日报”。保留原始来源可用于内部核对，不要把“检察日报-理论版”写进固定 citation。

## 回答格式

### 检索到可支撑内容时

```markdown
依据内置检答网集萃检索结果，可以参考如下意见：

1. [在检答网答疑支持范围内直接回答用户问题。]
   引用：检答网集萃第X批·问题Y（检察日报，YYYY-MM-DD）

2. [如有适用前提、办理要点或例外，继续说明。]
   引用：检答网集萃第X批·问题Y（检察日报，YYYY-MM-DD）

检索说明：本回答仅依据内置检答网集萃中检索到的内容整理，不等同于完整法律法规、司法解释或案例检索意见。
```

多个问题块共同支撑时，每个结论后分别引用。不要只在文末堆放引用。

### 未检索到可支撑内容时

```markdown
未在检答网集萃中检索到相关内容，因此不能基于检答网集萃给出确定答复。

如你需要，我可以在你授权后改用法律法规、司法解释、指导性案例或其他公开资料继续检索。
```

### 弱相关时

```markdown
检索到相近内容，但未检索到能直接支撑该问题的答疑内容。

相近内容是：[说明相近问题和边界]
引用：检答网集萃第X批·问题Y（检察日报，YYYY-MM-DD）

因此，不能把上述答疑直接扩展为对你问题的确定答复。
```

## 禁止事项

- 不得不检索就直接回答检答网集萃观点。
- 不得编造或猜测批次号、问题号、发布日期、来源、专家姓名。
- 不得把一般法律知识伪装成检答网答疑意见。
- 不得把弱相关检索结果强行写成直接答案。
- 未检索到时，不得为了“有帮助”而补写未经检答网集萃支持的结论。

## 维护与更新

如新增第141批以后资料，应按以下顺序维护：

1. 将新增 Markdown 放入 `references/batches/`。
2. 进入技能目录后运行：`python scripts/build_index.py --strict`。
3. 用典型问题运行 `python scripts/search_jiandawang.py ...` 检查命中效果。
4. 检查新增问题的 citation 是否仍符合固定格式。
5. 必要时更新 `references/legal-term-map.md` 和测试用例。
